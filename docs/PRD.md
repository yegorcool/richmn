# PRD: Richmn — казуальная merge-игра для Telegram Mini App

## 1. Обзор продукта

### 1.1 Концепция

Richmn — казуальная merge-2 игра, в которой игрок объединяет предметы на игровом поле, выполняет заказы и обустраивает уютное пространство. Игра предлагает множество тематических коллекций предметов (кофейня, выпечка, инструменты и др.), регулярные ивенты и расслабляющий, но затягивающий геймплей без давления таймерами.

### 1.2 Платформа

Telegram Mini App (WebView). HTML5 + JavaScript. Полноэкранный портретный режим.

### 1.3 Целевая аудитория

| Параметр | Значение |
|---|---|
| Пол | Женщины (80%+) |
| Возраст | 35–55 лет |
| География | Россия, СНГ (первичный рынок); Tier-1 (масштабирование) |
| Паттерн | Играют в транспорте, перед сном, в перерывах. Сессии 5–15 минут, 3–5 раз в день |
| Мотивация | Расслабление, коллекционирование, чувство прогресса, уютная атмосфера |

### 1.4 Бизнес-модель

Доход исключительно от рекламы. IAP отсутствуют. Ключевая метрика — суммарное время, проведённое пользователем в игре (time spent), которое напрямую коррелирует с количеством показов рекламы.

### 1.5 Рекламная интеграция

Интеграция через Monetag SDK (npm-пакет `monetag-tg-sdk`). Три формата:

| Формат | Когда показывается | Лимит |
|---|---|---|
| **Rewarded Interstitial** | Игрок нажимает кнопку «Получить заряды» / «Ускорить» / «Открыть сундук» | До 8 просмотров в день |
| **In-App Interstitial** | При возвращении из ивента на основное поле, при смене локации, после завершения заказа (не чаще 1 раза в 3 минуты) | Без жёсткого лимита, контролируется частотой |
| **Rewarded Popup** | Удвоение ежедневной награды, бонусный предмет | До 3 раз в день |

## 2. Core Loop

### 2.1 Схема основного игрового цикла

```
Генератор создаёт предметы
        ↓
Игрок перетаскивает предмет на такой же (merge-2)
        ↓
Получается предмет следующего уровня
        ↓
Предмет высокого уровня сдаётся в заказ
        ↓
Награда: монеты, опыт, ресурсы декора
        ↓
Разблокировка новых генераторов / тематик / декора
        ↓
Новые заказы → цикл повторяется
```

### 2.2 Игровое поле

> **✅ Реализовано.** Поле рендерится через PixiJS v8 (Canvas). Сетка 6×8, размер клетки 56×56px, зазор 4px. Фон — пастельные скруглённые прямоугольники. Состояние поля хранится серверно в таблицах `items` и `generators` (позиция `grid_x`, `grid_y`). При первом входе нового пользователя `GameInitService` автоматически размещает 3 стартовых генератора на позициях (1,5), (3,5), (5,5).

- Сетка 6×8 клеток (48 слотов) — оптимально для вертикального экрана телефона
- Предметы занимают 1 клетку
- Drag-and-drop для перемещения и объединения
- Тактильная обратная связь (haptic feedback через Telegram API) при успешном merge
- Удовлетворяющие анимации при объединении: частицы, свечение, лёгкий «поп»

### 2.3 Механика Merge-2

> **✅ Реализовано.** `MergeService` валидирует и выполняет merge в транзакции БД. Chain merge реализован рекурсивно (проверка 4 соседних клеток). За каждый merge начисляется опыт: `уровень × 5 + (длина_цепочки - 1) × 10`. При повышении уровня проверяется порог `level × 100 + level² × 10`. Фронтенд анимирует merge (shrink → flash → pop-in), chain combo (текст «x2!», «x3!»), и рассыпание частиц.

- Объединение двух одинаковых предметов = один предмет следующего уровня
- Каждый merge расходует 1 заряд энергии
- Chain merge: если результат merge оказывается рядом с ещё одним таким же предметом, срабатывает автоматический chain → бонусные очки, визуальный эффект «комбо»
- Merge возможен только для предметов одной цепочки и одного уровня

### 2.4 Генераторы

> **✅ Реализовано.** `GeneratorService` обрабатывает тап: проверяет готовность, энергию, ищет свободную клетку (приоритет — соседние 8 клеток), списывает энергию, создаёт предмет со взвешенным случайным уровнем (1..min(level, 3)). Для chargeable: декремент зарядов, установка cooldown при исчерпании. Для cooldown: установка cooldown на каждый тап. Merge двух генераторов одного уровня → генератор N+1 с увеличенными зарядами. Настройки генератора (`generation_limit`, `generation_timeout_seconds`, `energy_cost`) конфигурируются на уровне каждого экземпляра и темы.

Генераторы — источники новых предметов на поле. Два типа:

| Тип | Механика | Пример |
|---|---|---|
| **Заряжаемый** | 5 тапов для выдачи предмета. После исчерпания — перезарядка (таймер или merge генератора) | Кофемашина, духовка |
| **Кулдаунный** | Выдаёт 1 предмет бесплатно каждые N минут. Не расходует заряды при тапе | Грядка, почтовый ящик |

Генераторы можно улучшать через merge (два генератора одного уровня → генератор следующего уровня, выдающий предметы более высокого уровня).

### 2.5 Заказы

- Панель заказов отображается в верхней части экрана (до 3 активных заказов)
- Каждый заказ привязан к персонажу-заказчику (см. раздел 6 — Kill Feature). Аватар персонажа отображается на карточке заказа, а персонаж комментирует действия игрока ситуативными репликами
- Заказ содержит 1–3 предмета определённого уровня из одной или нескольких тематик
- Выполнение заказа: перетащить нужный предмет на карточку заказа
- Награда: монеты, опыт, ресурсы для декора, иногда — сундуки с предметами
- Новый заказ появляется через 30 секунд после выполнения предыдущего
- Сложность заказов масштабируется с уровнем игрока

### 2.6 Сундуки

- Выпадают из заказов, ивентов, ежедневных наград
- Открываются по таймеру (15 мин / 1 час / 4 часа) или мгновенно за просмотр rewarded ad
- Содержат: предметы, генераторы, монеты, бустеры

## 3. Система энергии (заряды)

> **✅ Реализовано.** `EnergyService` управляет энергией: авто-восстановление (1 заряд / 3 мин, вычисляется при каждом запросе), расход при merge (source: `merge`) и при тапе генератора (source: `generator`), пополнение через рекламу и бонусы. Все операции логируются в `energy_logs`. Фронтенд проверяет энергию перед тапом генератора — при нехватке показывает тряску генератора и открывает модал пополнения.

Заряды — основной ограничитель сессии и главный инструмент монетизации через рекламу.

### 3.1 Параметры

| Параметр | Значение |
|---|---|
| Максимум зарядов | 50 |
| Стоимость одного merge | 1 заряд |
| Скорость восстановления | 1 заряд / 3 минуты |
| Полное восстановление | 2,5 часа |
| Заряды за rewarded ad | +10 зарядов |
| Лимит просмотров рекламы в день | 5 |
| Бонус за ежедневный вход | +15 зарядов |
| Бонус за streak (7 дней подряд) | +50 зарядов (полный запас) |

### 3.2 UX энергии

- Шкала зарядов — всегда видна в верхней части экрана (иконка молнии + число)
- При 10 и менее зарядах — мягкая подсветка кнопки «Получить заряды»
- При 0 зарядах — всплывающее окно с вариантами:
  - «Посмотреть рекламу» (+10 зарядов) — основной CTA
  - «Подождать» (таймер до следующего заряда)
  - «Пригласить друга» (+20 зарядов за каждого нового игрока)
- Заряды НЕ тратятся на: тапы по генераторам, перемещение предметов, сдачу заказов, открытие сундуков — только на merge

### 3.3 Обоснование дизайна

Баланс рассчитан так, что средняя сессия до исчерпания зарядов длится 8–12 минут (50 merge при ~10 сек на merge с учётом перемещений и тапов). С учётом rewarded ads — до 18–20 минут. Восстановление за 2,5 часа позволяет играть 4–5 сессий в день, формируя привычку возвращаться.

- Максимальный (10-й) уровень — всегда с золотым акцентом и эффектом свечения
- Все предметы должны быть легко различимы на маленьком экране телефона

## 5. Мета-прогресс

### 5.1 Система уровней игрока

- Опыт начисляется за: выполнение заказов, merge предметов высокого уровня, завершение ивентов
- Каждый новый уровень разблокирует: новые тематики, слоты заказов, декоративные элементы
- Всего 50 уровней в базовой версии

### 5.2 Декорирование

Мета-слой вдохновлён Gossip Harbor и Merge Mansion: игрок обустраивает уютное пространство.

**Локации** (разблокируются последовательно):

| # | Локация | Разблокировка | Элементов декора |
|---|---|---|---|
| 1 | Кухня | Старт | 12 |
| 2 | Гостиная | Уровень 6 | 15 |
| 3 | Сад | Уровень 12 | 18 |
| 4 | Спальня | Уровень 18 | 14 |
| 5 | Терраса | Уровень 25 | 16 |
| 6 | Мастерская | Уровень 32 | 12 |

Каждый элемент декора имеет 3 варианта стиля (игрок выбирает). Ресурсы для декора добываются из заказов и ивентов. Установка декора сопровождается приятной анимацией преображения.

### 5.3 Сюжет

- Лёгкий ненавязчивый нарратив: главная героиня обустраивает дом и открывает свой магазинчик
- Сюжет подаётся через короткие диалоги (2–4 реплики) между персонажами перед разблокировкой новой локации или тематики
- Персонажи подробно описаны в разделе 6 (Kill Feature)
- Тон: тёплый, позитивный, без конфликтов и драмы

### 5.4 Коллекция

- Альбом всех обнаруженных предметов (по тематикам)
- За полное заполнение цепочки — награда (заряды, уникальный декор)
- Экран статистики: всего merge, самый длинный streak, любимая тематика

## 6. Kill Feature: живые персонажи-заказчики

Ключевое отличие от конкурентов — заказчики не просто безликие карточки заказов, а живые персонажи с уникальным характером, которые реагируют на действия игрока ситуативными репликами. Это создаёт эмоциональную привязку, превращает рутинный merge в социальное взаимодействие и значительно увеличивает время сессии: игрок задерживается, чтобы «пообщаться» с любимым персонажем.

### 6.2 Система реплик

Реплики выбираются движком из базы данных по набору условий. Каждая реплика хранится как запись с привязкой к персонажу, триггеру и контексту.

#### Схема записи реплики в БД

```
{
  "id":            "marina_order_complete_fast_03",
  "character_id":  "marina",
  "trigger":       "order_complete",
  "conditions": {
    "speed":       "fast",           // fast | normal | slow
    "order_level": "any",           // 1-10 | "high" | "any"
    "streak":      ">=3",           // текущий streak заказов подряд
    "mood":        "any",           // happy | neutral | impatient | any
    "time_of_day": "any",           // morning | afternoon | evening | night | any
    "theme":       "coffee",        // тематика заказа или "any"
    "first_time":  false            // первый заказ этого персонажа
  },
  "text":          "Ого, как быстро! Ты прямо бариста от бога!",
  "priority":      70,
  "max_shows":     3,
  "cooldown_hours": 24
}
```

#### Триггеры реплик

| Триггер | Момент срабатывания | Частота |
|---|---|---|
| `order_appear` | Персонаж появляется с новым заказом | Каждый новый заказ |
| `order_complete` | Игрок сдал заказ полностью | Каждое выполнение |
| `order_partial` | Игрок сдал часть составного заказа | При каждой частичной сдаче |
| `order_waiting_long` | Заказ висит больше 5 минут | Однократно на заказ |
| `order_waiting_very_long` | Заказ висит больше 15 минут | Однократно на заказ |
| `merge_nearby` | Игрок сделал merge рядом с персонажем (на поле) | Не чаще 1 раза в 30 сек |
| `high_level_item` | На поле появился предмет 8+ уровня | Каждое появление |
| `energy_depleted` | У игрока закончились заряды | Однократно |
| `player_return` | Игрок вернулся в игру после паузы | При каждом входе |
| `event_start` | Начался ивент | Однократно |
| `chain_merge` | Игрок сделал chain-merge (3+ комбо) | Каждый chain |
| `idle_on_field` | Игрок не делает действий 20+ секунд | Однократно за период |
| `wrong_merge_attempt` | Игрок пытается объединить несовместимые предметы | Не чаще 1 раза в минуту |

#### Условия выбора (conditions)

| Условие | Возможные значения | Описание |
|---|---|---|
| `speed` | `fast`, `normal`, `slow` | Насколько быстро игрок выполнил заказ (относительно среднего времени) |
| `order_level` | `1`–`10`, `low`, `mid`, `high`, `any` | Уровень предметов в заказе |
| `streak` | Числовое выражение (`>=3`, `==1`, `>=10`) | Сколько заказов подряд выполнено без паузы |
| `mood` | `happy`, `neutral`, `impatient`, `any` | Текущее настроение персонажа (зависит от времени ожидания) |
| `time_of_day` | `morning`, `afternoon`, `evening`, `night`, `any` | Реальное время суток игрока |
| `theme` | ID тематики или `any` | Тематика предметов в заказе |
| `first_time` | `true`, `false` | Первый ли это заказ у данного персонажа |
| `player_level` | Числовое выражение | Уровень игрока |
| `relationship` | `new`, `familiar`, `loyal` | Сколько заказов суммарно выполнено для персонажа (0–5 / 6–30 / 31+) |

#### Алгоритм выбора реплики

1. Отфильтровать реплики по `character_id` + `trigger`
2. Отсечь реплики, не прошедшие по `conditions` (все условия кроме `any` должны совпадать)
3. Отсечь реплики, достигшие `max_shows` или на кулдауне (`cooldown_hours`)
4. Из оставшихся выбрать с наивысшим `priority`
5. При равном приоритете — случайный выбор
6. Если ничего не нашлось — fallback: выбрать из пула generic-реплик персонажа

### 6.4 Настроение персонажа (mood)

Каждый персонаж имеет динамическое настроение, которое меняется в зависимости от действий игрока и влияет на выбор реплик.

| Состояние | Условие перехода | Визуальное отображение |
|---|---|---|
| `happy` | Заказ выполнен быстро, streak >= 3, chain merge | Улыбка, сердечки/звёзды над персонажем |
| `neutral` | Начальное состояние, обычный темп | Спокойное выражение |
| `impatient` | Заказ ждёт > 5 мин | Нахмуренные брови, «...» над головой |
| `delighted` | Получен предмет 9–10 уровня, первый заказ | Восторг, анимация прыжка, конфетти |

Настроение сбрасывается в `neutral` при получении нового заказа.

### 6.5 Отношения (relationship level)

Суммарное количество выполненных заказов для персонажа формирует уровень отношений, который открывает новые реплики и визуальные награды.

| Уровень | Заказов выполнено | Эффект |
|---|---|---|
| `new` | 0–5 | Базовые реплики знакомства |
| `familiar` | 6–30 | Персонаж обращается по имени (если задано), реплики теплее |
| `loyal` | 31+ | Уникальные реплики, персонаж дарит бонусные предметы каждый 10-й заказ, особая рамка на аватаре |

### 6.6 UX отображения реплик

- Реплика появляется в speech bubble над аватаром персонажа в панели заказов
- Длительность показа: 3 секунды (или до тапа)
- Не более 1 реплики одновременно (очередь с приоритетом, если несколько триггеров сработали)
- Реплики НЕ блокируют геймплей — игрок может продолжать играть
- Анимация появления: мягкий fade-in + лёгкий bounce bubble
- Тап по реплике скрывает её

### 6.7 Контентные требования

| Параметр | Значение |
|---|---|
| Персонажей на старте (MVP) | 3 (Марина, Бабушка Зина, Кот Барсик) |
| Персонажей всего | 11 |
| Реплик на персонажа | 80–120 |
| Общий объём реплик в БД | ~1 000 |
| Уникальных триггеров | 13 |
| Комбинаций условий на триггер | 5–15 |
| Fallback-реплик на персонажа | 10 (на случай отсутствия подходящей) |

### 6.8 Влияние на retention и монетизацию

- **Эмоциональная привязка**: игроки возвращаются «поговорить» с любимым персонажем — увеличивает D7/D30 retention
- **Коллекционирование реплик**: разблокировка уникальных реплик на уровне `loyal` мотивирует повторные заказы
- **Продление сессии**: реплики при `idle_on_field` и `energy_depleted` мягко удерживают игрока на экране, увеличивая шанс просмотра rewarded ad
- **Социальный шеринг**: забавные реплики кота Барсика — повод поделиться скриншотом, органический рост
- **Рекламная точка**: при `energy_depleted` реплика персонажа с предложением «отдохнуть» подводит к кнопке «Посмотреть рекламу» — ожидаемый рост viewer rate на 5–10%

## 7. Система ивентов

### 7.1 Типы ивентов

#### Еженедельный тематический ивент

| Параметр | Значение |
|---|---|
| Длительность | 5 дней (пн–пт) |
| Механика | Отдельное мини-поле 5×6 с уникальной тематической цепочкой (например, «Японская кухня», «Рождество», «Пляж») |
| Энергия | Общая с основным полем |
| Награды | Эксклюзивные декоративные предметы (недоступны в основной игре), заряды, сундуки |
| Прогресс | Шкала очков ивента с 10 порогами наград |

#### Daily Challenge (ежедневное задание)

| Параметр | Значение |
|---|---|
| Длительность | 24 часа |
| Механика | 3 задания разной сложности (лёгкое / среднее / сложное) |
| Примеры заданий | «Выполни 5 заказов», «Сделай 30 merge», «Получи предмет 7 уровня» |
| Награды | Лёгкое: 5 зарядов. Среднее: 10 зарядов + сундук. Сложное: 20 зарядов + эксклюзивный предмет |
| Бонус за 3 из 3 | Супер-сундук |

#### Сезонные ивенты

Крупные ивенты, привязанные к праздникам и временам года:

| Сезон | Ивент | Уникальная цепочка |
|---|---|---|
| Зима | Новый год | Ёлочные игрушки (10 уровней) |
| Зима | 14 февраля | Валентинки и сладости |
| Весна | 8 марта | Цветы и подарки |
| Весна | Пасха | Яйца и куличи |
| Лето | Каникулы | Пляжные аксессуары |
| Осень | Хэллоуин | Тыквы и сладости |
| Осень | Урожай | Осенние фрукты и овощи |

Длительность: 7–14 дней. Включают отдельное поле, сюжетную линию, лидерборд среди друзей.

#### Streak-бонусы (ежедневный вход)

| День | Награда |
|---|---|
| 1 | 10 зарядов |
| 2 | Малый сундук |
| 3 | 15 зарядов |
| 4 | Средний сундук |
| 5 | 20 зарядов + генератор |
| 6 | Большой сундук |
| 7 | 50 зарядов + эксклюзивный декор |

После 7-го дня цикл повторяется с улучшенными наградами (второй круг дороже первого).

### 7.2 Календарь ивентов

Расписание обеспечивает, что в любой момент активны минимум 2 типа ивентов:
- Daily Challenge — всегда
- Еженедельный ивент — пн–пт
- Сезонный ивент — 3–4 раза в сезон

Это создаёт постоянное ощущение «есть чем заняться» и причину вернуться.

## 8. Монетизация: стратегия максимизации времени в игре

### 8.1 Философия

Игра не продаёт ничего за деньги. Единственный источник дохода — реклама. Задача дизайна — максимизировать:
1. **Retention** (D1, D7, D30) — чем дольше игрок в игре, тем больше рекламы он посмотрит за жизнь
2. **Количество сессий в день** — больше возвратов = больше interstitial
3. **Длительность сессии** — дольше играет = больше точек для rewarded ads
4. **Viewer rate** — процент игроков, добровольно смотрящих rewarded ads

### 8.2 Точки показа рекламы

| Точка | Формат | Триггер | Ожидаемый Viewer Rate |
|---|---|---|---|
| Дополнительные заряды | Rewarded | 0 зарядов или кнопка в UI | 70–85% |
| Ускорение генератора | Rewarded | Таймер перезарядки генератора | 40–55% |
| Мгновенное открытие сундука | Rewarded | Тап по сундуку с таймером | 50–65% |
| Удвоение награды за заказ | Rewarded | Попап после выполнения заказа | 35–50% |
| Удвоение ежедневной награды | Rewarded Popup | Экран ежедневной награды | 55–70% |
| Бонусный предмет ивента | Rewarded Popup | Экран прогресса ивента | 30–45% |
| Переход между экранами | Interstitial | Из ивента → основное поле, из декора → поле | 100% (принудительно) |
| Возвращение в игру | Interstitial | Открытие Mini App после паузы >5 мин | 100% (принудительно) |

### 8.3 Ограничения и антифрод

- Interstitial: не чаще 1 раза в 3 минуты
- Rewarded: суммарно не более 12 просмотров в день (по всем точкам)
- Preload рекламы при запуске для мгновенного показа
- Graceful fallback: если реклама не загрузилась — дать награду бесплатно (лучше не раздражать игрока)

### 8.4 Прогноз доходности

При целевых показателях (DAU 50K, 3,5 сессий/день, 4,2 rewarded + 2,1 interstitial на сессию):

| Метрика | Значение |
|---|---|
| Rewarded views / день | ~735K |
| Interstitial views / день | ~367K |
| Средний eCPM (CIS) | $6 (rewarded), $3 (interstitial) |
| Ежедневный доход | ~$5 500 |
| Месячный доход | ~$165 000 |

## 9. Retention-механики

### 9.1 Формирование привычки

| Механика | Как работает | Целевой эффект |
|---|---|---|
| **Энергия / заряды** | Заканчиваются за 8–12 мин, восстанавливаются за 2,5 ч | Возврат 3–5 раз в день |
| **Кулдаунные генераторы** | Бесплатный предмет готов через 30–60 мин | Причина вернуться «забрать» |
| **Таймеры сундуков** | 15 мин / 1 ч / 4 ч | Микро-возвраты |
| **Daily Challenge** | Обновляется раз в сутки | Ежедневный визит |
| **Streak-награды** | Растущая ценность непрерывного входа | Страх потерять streak |
| **Персонажи-заказчики** | Уникальные реплики, рост отношений, бонусы за лояльность | Эмоциональная привязка, желание «поговорить» |

### 9.2 Уведомления через Telegram-бота

Поскольку Mini App не поддерживает push-нотификации, используем сообщения бота:

| Триггер | Сообщение | Задержка |
|---|---|---|
| Заряды восстановлены | «Заряды полны! Самое время продолжить» | Через 2,5 ч после ухода |
| Сундук готов | «Твой сундук открылся! Забери награду» | По таймеру сундука |
| Генератор готов | «Кофемашина готова! Новые предметы ждут» | По кулдауну |
| Daily Challenge обновился | «Новые задания дня! Заходи за наградами» | 10:00 каждый день |
| Streak под угрозой | «Не забудь зайти! Твой streak уже N дней» | 20:00, если не заходил сегодня |
| Новый ивент | «Начался ивент "Пляжный сезон"! Собирай ракушки» | При запуске ивента |

Частота: не более 3 сообщений в день. Игрок может настроить/отключить уведомления.

### 9.3 Социальные механики

- **Приглашение друзей**: реферальная ссылка через Telegram. +20 зарядов приглашающему и приглашённому
- **Лидерборд друзей**: по очкам за текущий ивент (видим только друзей, играющих в игру)
- **Подарки**: раз в день можно отправить другу 5 зарядов (и получить столько же в ответ)
- **Совместные цели**: «Вместе с друзьями выполните 1000 заказов на этой неделе» → групповая награда

### 9.4 Антиотток

- Если игрок не заходил 3+ дня — бонус возвращения (полные заряды + сундук)
- Адаптивная сложность заказов: если игрок буксует — упростить следующие 2–3 заказа
- Новый контент (тематики, локации) распределён равномерно по прогрессу, чтобы всегда была ближайшая цель

## 10. UX и дизайн интерфейса

### 10.1 Принципы

- **Никакого давления**: нет обратных таймеров на заказах, нет штрафов за бездействие
- **Мгновенная обратная связь**: каждое действие сопровождается анимацией и звуком
- **Одна рука**: все элементы управления доступны большим пальцем
- **Крупные элементы**: минимальный размер тапабельной области — 44×44 pt
- **Чистота экрана**: основной экран — только поле, заказы и энергия. Всё остальное — через нижнюю навигацию

### 10.2 Структура экранов

├──────────────────────────────────┤
│                                  │
│                                  │

│                                  │
│                                  │

### 10.3 Визуальный стиль

- Палитра: пастельные тона — мятный, лавандовый, персиковый, кремовый
- Фон: мягкий градиент или текстура дерева/ткани
- Шрифт: скруглённый sans-serif, крупный (мин. 14pt)
- Иконки: flat с мягкими тенями, скруглённые углы
- Анимации: bouncing, мягкие easing-функции, без резких движений
- Звуки: приятные «поп», «звон», мелодичные нотификации. Фоновая музыка — lofi / jazz café (отключаемая)

## 11. Технические требования и ограничения

### 11.1 Платформа Telegram Mini App

| Аспект | Детали |
|---|---|
| Рендеринг | HTML5 Canvas (PixiJS или Phaser) в Telegram WebView |
| Хранение прогресса | Серверная БД (Telegram Cloud Storage: 1 MB — недостаточно для полного сохранения). Авторизация через `initData` |
| Фоновая работа | Не поддерживается. Кулдауны и таймеры вычисляются серверно |
| Уведомления | Через Telegram Bot API (`sendMessage`). Mini App не имеет push |
| Реклама | Monetag SDK (`monetag-tg-sdk`). Preload при запуске |
| Haptic Feedback | `Telegram.WebApp.HapticFeedback` API |
| Полный экран | `Telegram.WebApp.expand()` при запуске |
| Оплата | Не требуется (нет IAP). Резерв: Telegram Stars для будущих косметических покупок |

### 11.2 Производительность

- Целевой FPS: 60 на устройствах 2020+ года
- Размер первоначальной загрузки: < 3 MB (остальные ассеты — lazy load)
- Время старта: < 2 секунды до интерактивного состояния
- Оффлайн: базовый merge работает оффлайн, синхронизация при восстановлении связи

### 11.3 Бэкенд

- Серверное хранение состояния игры (прогресс, инвентарь, энергия, таймеры)
- Серверная валидация merge-операций для защиты от читов
- WebSocket для real-time обновлений (ивент-лидерборды, подарки друзей)
- REST API для CRUD-операций (заказы, декор, прогресс)

### 12.1 Ключевые метрики

| **Daily time spent** | 35–50 мин | Ключевая метрика для ad revenue |
| **Rewarded ad viewer rate** | ≥ 60% | Доля игроков, смотрящих хотя бы 1 rewarded ad в день |
| **Avg rewarded views / DAU** | ≥ 4 | Среднее количество просмотров rewarded ads на игрока |

## Приложение А. Глоссарий

| Термин | Определение |
|---|---|
| **Merge-2** | Механика объединения двух одинаковых предметов в один следующего уровня |
| **Цепочка (Chain)** | Последовательность предметов от 1-го до максимального уровня в одной тематике |
| **Генератор** | Объект на поле, создающий новые предметы при тапе или по таймеру |
| **Заряд** | Единица энергии, расходуемая при каждом merge |
| **Chain merge** | Автоматический merge, срабатывающий когда результат оказывается рядом с идентичным предметом |
| **Viewer rate** | Процент игроков, просмотревших хотя бы одну rewarded ad за сессию |
| **eCPM** | Effective Cost Per Mille — доход за 1000 показов рекламы |
| **Streak** | Серия дней непрерывного входа в игру |
| **DAU** | Daily Active Users — ежедневная активная аудитория |
| **Персонаж-заказчик** | NPC с уникальным характером, привязанный к тематике. Выдаёт заказы и комментирует действия игрока |
| **Триггер реплики** | Игровое событие, при котором движок ищет подходящую реплику для персонажа |
| **Mood** | Текущее настроение персонажа (happy / neutral / impatient / delighted), влияющее на выбор реплик |
| **Relationship** | Уровень отношений игрока с персонажем (new / familiar / loyal), растёт с количеством выполненных заказов |
